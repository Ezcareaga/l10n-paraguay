## 1. `lxml` — generación y parsing XML

- **Versión**: `>=4.9` (la 5.x también funciona)
- **URL**: https://lxml.de/
- **Uso en proyecto**: construir el XML del Documento Electrónico SIFEN, parsear
  respuestas SOAP de DNIT, validar contra XSD oficial.
- **Por qué lxml y no xml.etree del stdlib**: necesitamos XPath, namespaces,
  XSD validation y firma XAdES — todo eso lxml lo soporta, stdlib no.

### Snippets típicos

```python
from lxml import etree

# Construir el XML del DE
nsmap = {None: 'http://ekuatia.set.gov.py/sifen/xsd'}
root = etree.Element('rDE', nsmap=nsmap)
de = etree.SubElement(root, 'DE')
de.set('Id', invoice.l10n_py_cdc)
etree.SubElement(de, 'dVerFor').text = '150'
# ... resto de los grupos AA-H

xml_bytes = etree.tostring(
    root,
    pretty_print=False,        # SIFEN: sin pretty-print
    xml_declaration=True,
    encoding='UTF-8',
)

# Parsear respuesta SOAP
response_tree = etree.fromstring(response_xml_bytes)
codigo = response_tree.find('.//{http://ekuatia.set.gov.py/sifen/xsd}dCodResp')
print(codigo.text)  # '0260' si aprobado

# Validar contra XSD
xsd_doc = etree.parse('schemas/siRecepDE_v150.xsd')
schema = etree.XMLSchema(xsd_doc)
schema.assertValid(root)  # raises XMLSyntaxError si no valida
```

### Gotchas

- SIFEN exige **sin prefijos en namespace** — el `nsmap={None: '...'}` es la forma
  de tener namespace por defecto.
- SIFEN exige **sin whitespace** entre etiquetas — pasar `pretty_print=False`.
- Encoding UTF-8 sin BOM — `xml_declaration=True` lo hace bien.

## 2. `signxml` — firma XAdES-BES

- **Versión**: `>=3.2`
- **URL**: https://github.com/XML-Security/signxml
- **Uso en proyecto**: firma digital XMLDSig del DE con el CCFE (PKCS#12) del
  contribuyente.
- **Alternativa**: `cryptography` + `xmlsec` (más bajo nivel, más control —
  necesario si signxml no soporta algún detalle SIFEN).

### Snippets típicos

```python
from signxml import XMLSigner, methods
from cryptography.hazmat.primitives.serialization.pkcs12 import load_pkcs12

# Cargar PKCS#12
with open('ccfe.p12', 'rb') as f:
    pkcs = load_pkcs12(f.read(), password=ccfe_password.encode())

# Firmar
signer = XMLSigner(
    method=methods.enveloped,
    signature_algorithm='rsa-sha256',
    digest_algorithm='sha256',
    c14n_algorithm='http://www.w3.org/TR/2001/REC-xml-c14n-20010315',
)
signed_root = signer.sign(
    root,                              # lxml Element
    key=pkcs.key,
    cert=pkcs.cert.certificate,
    reference_uri=f'#{cdc}',           # SIFEN: CDC precedido por #
)
signed_xml = etree.tostring(signed_root, encoding='UTF-8', xml_declaration=True)
```

### Gotchas

- **SIFEN** especifica XAdES-BES enveloped. signxml hace eso por default.
- El `reference_uri='#<cdc>'` debe coincidir con `Id="<cdc>"` del nodo `<DE>`.
- Si la firma falla en SIFEN, comparar con un DE aprobado conocido — diff de
  canonicalization suele ser la causa.

## 3. `zeep` — cliente SOAP

- **Versión**: `>=4.2`
- **URL**: https://docs.python-zeep.org/
- **Uso en proyecto**: cliente SOAP 1.2 contra WSDL de SIFEN (siRecepDE,
  siRecepLoteDE, etc.) con mutual TLS.

### Snippets típicos

```python
from zeep import Client
from zeep.transports import Transport
from requests import Session
from requests_pkcs12 import Pkcs12Adapter

# Setup session con mutual TLS via PKCS#12
session = Session()
session.mount(
    'https://',
    Pkcs12Adapter(
        pkcs12_data=ccfe_bytes,
        pkcs12_password=ccfe_password,
    ),
)
transport = Transport(session=session, timeout=30)

# Cliente apuntando al WSDL test
wsdl = 'https://sifen-test.set.gov.py/de/ws/sync/recibe.wsdl'
client = Client(wsdl=wsdl, transport=transport)

# Llamada siRecepDE
response = client.service.siRecepDE(dId=str(uuid.uuid4()), xDE=signed_xml.decode())

# response es un dict-like
print(response['dCodRes'])     # '0260'
print(response['dMsgRes'])     # 'Aprobado'
```

### Gotchas

- SIFEN devuelve respuestas SOAP que pueden ser de un solo elemento — Zeep las
  envuelve en dicts; navegar con cuidado.
- Para `siRecepLoteDE`: comprimir el XML del lote y base64-encode ANTES de
  pasarlo.
- Timeouts: default de zeep es muy bajo (5s) — subir a 30s+ porque SIFEN test
  puede tardar.

## 4. `cryptography` — manejo PKCS12, AES, RSA, hashing

- **Versión**: `>=42.0`
- **URL**: https://cryptography.io/
- **Uso en proyecto**: cargar PKCS#12, encriptar el CCFE en BD con AES-256
  (Fernet), hashear (SHA-256 para el hash QR), generar bytes random para el
  código de seguridad del CDC.

### Snippets típicos

```python
# Cargar PKCS#12
from cryptography.hazmat.primitives.serialization.pkcs12 import load_pkcs12
with open('ccfe.p12', 'rb') as f:
    p12 = load_pkcs12(f.read(), password=b'mi-password')
private_key = p12.key
cert = p12.cert.certificate

# Extraer RUC del cert (SerialNumber para PJ)
ruc_in_cert = cert.subject.get_attributes_for_oid(
    x509.NameOID.SERIAL_NUMBER
)[0].value

# Encriptar CCFE con Fernet (AES-128 + HMAC)
from cryptography.fernet import Fernet
import os
key = os.environ['CCFE_ENCRYPTION_KEY'].encode()  # 32 bytes base64
f = Fernet(key)
encrypted_ccfe = f.encrypt(ccfe_bytes)

# Hashear para QR
from cryptography.hazmat.primitives import hashes
digest = hashes.Hash(hashes.SHA256())
digest.update(f"{cdc}|{csc}".encode())
hash_qr = digest.finalize().hex()
```

### Gotchas

- `load_pkcs12` falla con error críptico si la password es incorrecta — wrap en
  try/except y dar mensaje user-friendly.
- Para Fernet keys: generar UNA vez con `Fernet.generate_key()` y guardar en
  env var; **nunca regenerar en runtime**.

## 5. `qrcode` — generación de QR code para KuDE

- **Versión**: `>=7.4`
- **URL**: https://github.com/lincolnloop/python-qrcode
- **Uso en proyecto**: generar QR del KuDE con URL de consulta SIFEN.

### Snippets típicos

```python
import qrcode
import io
import base64

qr_url = (
    f"https://ekuatia.set.gov.py/consultas/qr?"
    f"nVersion=150&Id={cdc}&dFeEmiDE={fecha_emision_iso}"
    f"&dRucRec={ruc_receptor}&dTotGralOpe={total_general}"
    f"&dTotIVA={total_iva}&cItems={n_items}"
    f"&DigestValue={digest_value}&IdCSC={csc_id}&cHashQR={hash_qr}"
)

qr = qrcode.QRCode(
    version=None,                 # auto-tamaño
    error_correction=qrcode.constants.ERROR_CORRECT_M,
    box_size=10,
    border=2,
)
qr.add_data(qr_url)
qr.make(fit=True)

img = qr.make_image(fill_color="black", back_color="white")

# Para embedir en QWeb report:
buffer = io.BytesIO()
img.save(buffer, format='PNG')
qr_base64 = base64.b64encode(buffer.getvalue()).decode()

# En QWeb:
# <img t-att-src="'data:image/png;base64,%s' % qr_base64"/>
```

### Gotchas

- Para tickets POS (cinta de papel angosta), `box_size=6` y `border=1` queda
  bien.
- Para PDF carta, `box_size=10` y `border=2` es estándar.

## 6. `requests-pkcs12` — mutual TLS con PKCS#12 directo

- **Versión**: `>=1.21`
- **URL**: https://github.com/m-click/requests_pkcs12
- **Uso en proyecto**: agregar PKCS#12 como client cert al adapter de `requests`
  sin tener que separar `.p12` en `.key` + `.crt` (lo que Python's standard
  ssl exige).
- **Alternativa**: convertir `.p12` a `.pem` con openssl al subir el cert y
  usar `verify=True, cert=('cert.pem', 'key.pem')` — más fricción operativa.

### Snippets típicos (ver Zeep arriba — uso integrado)

```python
from requests import Session
from requests_pkcs12 import Pkcs12Adapter

session = Session()
session.mount(
    'https://sifen-test.set.gov.py',
    Pkcs12Adapter(pkcs12_data=ccfe_bytes, pkcs12_password=ccfe_password),
)

# Ahora cualquier request al sifen-test usa el cert
resp = session.post('https://sifen-test.set.gov.py/de/ws/sync/recibe.wsdl',
                    data=soap_envelope, headers={'Content-Type': 'application/soap+xml'})
```

## 8. Librerías que evaluamos pero NO usamos (y por qué)

| Librería                   | Por qué no                                                                                 |
| -------------------------- | ------------------------------------------------------------------------------------------ |
| `xmlsec` (Python bindings) | Mejor performance que signxml pero compilación nativa compleja en Windows; signxml alcanza |
| `suds-py3`                 | Cliente SOAP más viejo y menos mantenido que zeep                                          |
| `pyOpenSSL`                | Cryptography moderna preferida; pyOpenSSL en modo mantenimiento                            |
| `qrcode-terminal`          | Solo terminal; necesitamos PIL backend                                                     |
| `python-pkcs12`            | menos popular; requests-pkcs12 mejor integrado con requests/zeep                           |
