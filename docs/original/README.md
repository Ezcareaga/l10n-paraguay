### ⚠ Importante sobre los XSDs

Los archivos en `xsd/` provienen de los RARs descargados del portal DNIT
(`estructura_DE_xsd.rar`, `estructura_xml_DE.rar`) **descomprimidos el
2026-05-20**. Inspeccionando su contenido:

- `Estructura_DE_xsd.xml` (fecha 2018-02-07) es un XSD real pero **muy
  preliminar** — solo define los grupos básicos del `rDE` raíz. No es el
  schema v150 que valida SIFEN producción.
- `Extructura_xml_DE.xml` (fecha 2020-05-20) **NO es un schema**, es un
  ejemplo concreto de un DE serializado v150 con namespace
  `http://ekuatia.set.gov.py/sifen/xsd` y referencia a `siRecepDE_v150.xsd`.

Para la implementación real de `l10n_py_edi` se necesitan los XSDs canónicos
v150 (`siRecepDE_v150.xsd`, `Evento_v150.xsd`, `siTypes_v150.xsd`, etc.) que
NO están publicados directamente en el portal DNIT. Tres caminos:

1. **Solicitar a DNIT** vía `consultas.dnit@set.gov.py` (lo formal).
2. **Extraer de `jsifenlib`** (Roshka, Java) — repo público con los XSDs en
   `src/main/resources/`: https://github.com/roshka/jsifenlib
3. **Generar desde el Manual Técnico v150** — sección 9 detalla cada tipo
   con suficiente nivel para reconstruir el XSD. Más trabajoso pero garantiza
   alineación con la versión vigente.

### Procedimiento manual sugerido

1. Abrir https://www.dnit.gov.py/web/e-kuatia en un navegador
2. Navegar a "Normativas" → descargar Manual Técnico + Manual Especificaciones
3. Navegar a "Documentación Técnica" → descargar zip de XSDs
4. Descomprimir XSDs en `docs/original/xsd/`
5. Catálogos: probablemente en el zip de documentación técnica, o en
   "Tablas y Catálogos"
6. Para cada PDF descargado: ejecutar la skill `pdf-reading` para convertir a
   markdown:
   ```
   /pdf-reading docs/original/sifen_manual_tecnico_v150.pdf
   ```
   El output va a `docs/02_SIFEN_REFERENCIA_COMPLETA.md` como sección
   "Actualizaciones desde Manual Técnico DNIT".

### Alternativa: jsifenlib de Roshka

Si los XSDs no se pueden bajar del portal oficial, el repo Java `jsifenlib` de
Roshka (https://github.com/roshka/jsifenlib) los incluye en `src/main/resources/`.
Aunque jsifenlib es Java, los XSDs son agnósticos.
