### Change 4 — pylint_odoo `scripts/` exclude (DONE)

Added `exclude: ^scripts/.*\.py$` to BOTH `pylint_odoo` hook invocations
(informational + mandatory). Reason: `pylint_odoo`'s `walk_up()` enters
infinite recursion on Windows because `os.path.dirname("C:\\")` returns
`"C:\\"` (not `""`), triggered by any `.py` outside an addon directory.
`scripts/build_index.py` was the trigger. Exclude is Windows-pragmatic;
upstream `pylint_odoo` should fix the recursion guard, but until then we
side-step it.

### Change 5 — `oca-checks-odoo-module` deferred XML check (DONE — with deviation)

**Deviation from prompt:** the prompt suggested an `exclude:` block listing
the 7 XMLs. That approach **does not work** because
`oca-checks-odoo-module` scans the entire module directory once per
manifest file (not per-XML). Excluding XML files from the hook just removes
them from the hook's input set, but the hook still scans the module folder
from disk when it sees a `__manifest__.py`.

**Corrected fix:** pass `args: ["--disable=xml-deprecated-data-node"]` to
the hook — disables the specific check globally, leaving all other
`oca-checks-odoo-module` checks active. This is the canonical OCA pattern
(same shape as the existing `oca-checks-po: --disable=po-pretty-format`
right next to it).

Documented as **TD-004** in `BUGS_BACKLOG.md` (7 files affected, 6
auto-generated by `scripts/generate_module_data.py`, 1 hand-edited).
