### Read-only / structural hooks (FAILED ‚Äî escalation territory)

#### 1. `flake8` ‚Äî HOOK BROKEN (env-compat)

```
AttributeError: 'EntryPoints' object has no attribute 'get'
```

**Root cause:** flake8 v3.9.2 (pinned in `.pre-commit-config.yaml` line 68) is
incompatible with Python 3.12's `importlib.metadata`. The `.get()` call on
`EntryPoints` was removed in Python 3.12.

**Fix required:** bump pin to `flake8` v6.x or v7.x in `.pre-commit-config.yaml`.
This is a P1-A config file modification.

**Recommended pin:** `7.1.1` (current stable, OCA-aligned; l10n-brazil uses 6.1.0).

#### 2. `pylint_odoo` ‚Äî HOOK BROKEN (env-compat, Windows)

```
RecursionError: maximum recursion depth exceeded
File "...\pylint_odoo\misc.py", line 103, in walk_up
  return walk_up(os.path.dirname(path), filenames, top)
```

**Root cause:** `pylint_odoo`'s `walk_up()` looks for `__manifest__.py` walking
parent dirs. On Windows, `os.path.dirname("C:\\")` returns `"C:\\"` (not `""`),
causing infinite recursion. Triggered by `scripts/build_index.py` (which is
outside any addon).

**Fix required:** add `ignore-paths = scripts/` or `^scripts/` to BOTH
`.pylintrc` and `.pylintrc-mandatory`. This is a P1-A config file modification.

#### 3. `pylint_odoo` ‚Äî REAL FINDING (manifest dup key)

```
addons\l10n_py_account\__manifest__.py:3: W0109 Duplicate key 'auto_install' in dictionary
```

**Inspection:** `__manifest__.py` has `"auto_install": ["account"]` on line 20
AND `"auto_install": False` on line 37. Second silently wins. The `["account"]`
form is invalid for `auto_install` (it's a bool, not a list ‚Äî `depends:
["account"]` is the right key). Likely copy-paste artifact.

**Fix required:** delete line 20 (`"auto_install": ["account"]`). The
intent-preserving value is `False` per line 37. **THIS IS A REAL BUG worth
fixing inline (Rule 1).**

#### 4. `oca-checks-odoo-module` ‚Äî REAL FINDING (XML deprecation)

7 files flagged `xml-deprecated-data-node`:

- `addons/l10n_py_account/security/l10n_py_account_security.xml`
- `addons/l10n_py_base/data/l10n_latam_identification_type_data.xml`
- `addons/l10n_py_base/data/l10n_py_economic_activity_demo.xml`
- `addons/l10n_py_base/data/l10n_py_recipient_nature_data.xml`
- `addons/l10n_py_base/data/l10n_py_regime_data.xml`
- `addons/l10n_py_base/data/l10n_py_taxpayer_type_data.xml`
- `addons/l10n_py_base/data/res_country_state_data.xml`

**Root cause:** Odoo 18 deprecated nested `<data noupdate="1">` inside `<odoo>`.
Modern form: `<odoo noupdate="1">` (move `noupdate` to root element, delete
inner `<data>`).

**Fix required:** 6 of the 7 XMLs are auto-generated by
`scripts/generate_module_data.py` (line 44 has the template `<odoo><data
noupdate="1">`). Fix template + regenerate. The 7th (security XML) is
hand-edited.

**Estimated effort:** ~10 min generator fix + regen + visual diff check. But
data files load at module install ‚Äî D-07 AFTER must re-run to confirm tests
pass. **NOT trivial enough for P1-C semantic baseline scope.**

**Recommended disposition:** defer to BUGS_BACKLOG TD-003.
